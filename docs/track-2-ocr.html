Track 2 / Ruta Purista

OCR con Tesseract

De imagen a texto útil.

Abrir notebook en GitHub Abrir en Colab

Inspirado en Practical Natural Language Processing (O’Reilly), en practicalnlp.ai y en practical-nlp-code/Ch2. No existe afiliación oficial con esos recursos salvo que se indique explícitamente.

Qué vas a aprender

  • Qué hace Tesseract y dónde aporta valor frente a una extracción manual.
  • Cómo preparar una imagen para que el OCR lea mejor el texto.
  • Cómo evaluar el resultado y detectar errores típicos de OCR.
  • Cómo convertir texto desde imagen en un insumo más usable para NLP.

Por qué importa para datasets de NLP

Mucho texto útil no nace limpio: llega desde capturas, escaneos, documentos o fotos. OCR permite recuperar ese material para construir corpora, auditar datasets y ampliar fuentes de entrenamiento cuando el texto original no está disponible.

Dependencias y entorno

Parte Qué necesitás
Tesseract El binario del motor OCR instalado en el sistema.
Python wrapper La librería que llama al binario desde el notebook.
Colab / Linux Suelen ser más directos para instalar y probar Tesseract.
Windows Puede requerir una instalación manual del binario y ajustar la ruta.

Antes de abrir el notebook

  • Leé la historia completa: OCR no es solo correr una celda.
  • Confirmá que Tesseract esté instalado y accesible desde el entorno.
  • Si no subiste una imagen propia, el notebook genera automáticamente una imagen demo para que puedas arrancar sin trabarte.
  • Si querés, podés reemplazar esa demo con tu propia imagen subida al notebook antes de correr OCR.
  • Si el texto sale imperfecto, probá primero con la imagen y el preprocesamiento, no con atajos.
  • Recordá que esta ruta es más técnica que Track 1, no “mejor”: tiene otro foco de aprendizaje.

Notas de clase

OCRLeer bien empieza antes del reconocimiento

Si la imagen entra sucia, el OCR no la salva por magia. Iluminación, recorte y contraste cambian mucho el resultado.

EntornoBinario y wrapper son dos piezas distintas

Instalar el paquete de Python no siempre instala el motor. Para que funcione de verdad, el binario de Tesseract también tiene que estar disponible.

DatosTexto recuperado no es texto perfecto

El OCR produce ruido, omisiones y errores de segmentación. Después viene la limpieza, la validación y la decisión de qué sirve para NLP.
