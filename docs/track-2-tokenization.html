Track 2 / Ruta Purista

Tokenización y preparación lingüística

Del texto adquirido al texto preparado.

Abrir notebook en GitHub Abrir en Colab

Inspirado en Practical Natural Language Processing (O’Reilly), en practicalnlp.ai y en practical-nlp-code/Ch2. No existe afiliación oficial con esos recursos salvo que se indique explícitamente.

Qué vas a aprender

  • Cómo pasar de texto bruto a tokens útiles para análisis.
  • Cuándo conviene quitar stopwords y cuándo no conviene apurarse.
  • Qué cambia entre stemming y lematización.
  • Cómo POS tagging agrega contexto sintáctico al texto.

Por qué importa después de scraping y OCR

Primero conseguís texto: desde una página web o desde una imagen. Después empieza el trabajo fino. Tokenización, stopwords, stemming, lematización y POS tagging convierten ese texto en una base más estable para medir, comparar, etiquetar y preparar datasets de NLP con menos ruido.

Setup del entorno

  • El notebook usa recursos de NLTK: `punkt`, `punkt_tab`, `stopwords`, `wordnet`, `omw-1.4`, `averaged_perceptron_tagger` y `averaged_perceptron_tagger_eng`.
  • También descarga el modelo de spaCy `en_core_web_sm`.
  • Si una importación o recurso falla, volvé a correr la celda de setup y reiniciá el runtime si hace falta.

Antes de abrir el notebook

  • Leé la historia completa: el orden importa.
  • Entendé primero qué problema resuelve cada técnica; no son pasos decorativos.
  • Recordá que Track 2 profundiza más que Track 1: Módulo 1, pero no lo reemplaza.
  • Si el entorno cambia, repetí setup antes de seguir con el flujo principal.

Notas de clase

TokenizaciónSeparar bien ya es pensar bien

Si los tokens salen mal, todo lo demás se apoya sobre una base débil. La limpieza empieza en la segmentación.

NormalizaciónNo toda variación vale lo mismo

Stemming y lematización reducen variantes, pero no siempre conviene tratarlas igual. El criterio depende del objetivo.

POS taggingEl contexto cambia el significado

Ver la función gramatical ayuda a leer mejor el texto y a tomar decisiones más finas sobre análisis y features.
