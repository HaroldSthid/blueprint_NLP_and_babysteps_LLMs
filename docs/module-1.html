Módulo 1

Cómo una máquina lee texto.

En este primer módulo vas a seguir el recorrido de un notebook que enseña a un modelo a decidir si una reseña de película es positiva o negativa.

Notas de clase

Colab y runtimeEl entorno es temporal

Si Colab pide reiniciar o vuelve a cargar, no es un fracaso: el runtime se limpia. Guardá lo importante fuera de archivos temporales.

Limpieza de textoMenos ruido, más señal

Limpiar texto ayuda al modelo a comparar mejor: quitá HTML, puntuación y mayúsculas, y usá stopwords o lematización cuando el flujo lo pida.

Warnings y erroresNo todos los mensajes frenan igual

Un warning avisa; un traceback frena la ejecución. Leé el mensaje completo antes de seguir para no tapar la causa real.

TF-IDF y matriz de confusiónPasar de texto a señales

TF-IDF convierte texto en números útiles; la matriz de confusión muestra dónde el modelo acierta y dónde mezcla clases.

Objetivo del módulo

Entender cómo una máquina transforma texto en señales útiles, qué significa preparar ese material y por qué el camino incluye errores, reinicios y ajustes reales.

Antes de abrir el notebook

  • No necesitás saber programación avanzada.
  • Conviene leer las instrucciones con calma antes de ejecutar celdas.
  • Si algo no se entiende, anotá la duda en vez de saltarlo.
  • Recordá que este módulo es para mirar el proceso, no para memorizar todo.
  • Si estás en clase, seguí la guía de clase 1 para no saltearte el flujo.

Vocabulario clave

Término En simple
Texto Lo que escribimos o leemos: una frase, un comentario, una reseña.
Token Una pieza pequeña del texto, como una palabra o parte de una palabra.
Número La forma en que la computadora representa el texto para poder calcular con él.
Modelo El sistema que aprende patrones a partir de muchos ejemplos.
Predicción La respuesta que el modelo intenta dar a partir de lo que vio.

Camino del notebook

Pensalo como preparar a un estudiante para un examen importante: primero armamos el entorno, después practicamos, corregimos errores y recién al final pedimos una respuesta al modelo.

0

Preparar la mochila

Instalamos bibliotecas, resolvemos conflictos de versiones y, si hace falta, reiniciamos para limpiar el entorno.

1

Traer las reseñas

Intentamos cargar el dataset con tensorflow_datasets y, si falla, usamos la descarga y extracción directa como respaldo.

2

Limpiar las reseñas

Quitamos HTML y puntuación, pasamos a minúsculas, lematizamos y manejamos diccionarios faltantes o errores en reseñas puntuales.

3

Preparar las entradas

Convertimos cada reseña en una forma útil para el modelo y aceptamos que algunos casos necesiten correcciones extra.

4

Mirar y entender

Revisamos el balance de clases y generamos word clouds para ver qué palabras dominan el conjunto.

5

Entrenar el modelo

El modelo aprende patrones de sentimiento y luego lo evaluamos, con resultados cercanos al 88% de accuracy.

6

Guardar lo aprendido

Guardamos el modelo y el vectorizador para no repetir el trabajo desde cero.

7

Probar con reseñas nuevas

Clasificamos textos reales y resolvemos el detalle de convertir la salida flotante a entero para leer la confianza correctamente.

8

Entender por qué decidió eso

Miramos qué palabras empujan hacia lo positivo o lo negativo para explicar la predicción.

Acá, los errores y los reinicios no son fracaso: son parte del aprendizaje. Cada ajuste deja el camino más robusto y más fácil de repetir.
