Track 2 / Ruta Purista

Data Augmentation con NLPaug ModernLab

Entrenar con variaciones: preparar un modelo para texto imperfecto del mundo real.

Abrir notebook en GitHub Abrir en Colab

Inspirado en Practical Natural Language Processing (O’Reilly), en practicalnlp.ai y en practical-nlp-code/Ch2. No existe afiliación oficial con esos recursos salvo que se indique explícitamente.

Qué vas a aprender

  • Cómo generar variaciones de texto sin perder de vista el objetivo del dataset.
  • Cómo usar NLPaug para ruido de caracteres, sustituciones de palabras y ejemplos contextuales.
  • Cómo pensar la augmentación como una herramienta de robustez, no como un adorno.
  • Cómo leer el costo técnico de cada transformación antes de aplicarla en producción.

Por qué importa después de tokenización y limpieza

La augmentación funciona mejor cuando el texto ya pasó por tokenización y preprocesamiento básico. Primero ordenás la materia prima; después introducís variación con intención. Así evitás amplificar ruido, reducís errores obvios y entrenás al modelo para tolerar escritura real, abreviaturas, typos y cambios leves.

Cuándo ayuda y cuándo puede dañar

Ayuda cuando... Puede dañar cuando...
tenés pocos ejemplos y querés robustez frente a variaciones de escritura; el dataset ya está bastante sucio y la augmentación solo multiplica el ruido;
querés exponer al modelo a typos, sinónimos o pequeñas perturbaciones; la etiqueta depende de una palabra exacta y la transformación rompe el significado;
buscás mejorar generalización en un dominio estable; la tarea es sensible al contexto, a nombres propios o a matices semánticos finos.

Setup del entorno

  • Dependencias clave: `nlpaug`, `nltk`, `pandas`, `matplotlib` y `transformers`.
  • La parte contextual puede descargar `distilbert-base-uncased`; la primera corrida puede ser más lenta.
  • La limpieza opcional del runtime solo tiene sentido si estás resolviendo conflictos desechables de Colab.
  • Si cambian las dependencias, repetí la celda de setup antes de seguir con el laboratorio.

Antes de abrir el notebook

  • Leé la secuencia completa: el orden importa para no mezclar limpieza con augmentación.
  • Recordá que Track 2 es más profundo y técnico, pero no “mejor” que Track 1: Módulo 1.
  • Usá ModernLab como camino preferido; el notebook legacy queda como referencia histórica y comparación.
  • Si querés repetir el flujo, hacelo con intención: no toda transformación vale para todos los casos.

Notas de clase

RobustezVariar no es inventar sin criterio

La augmentación sirve para enseñar tolerancia a la forma, no para esconder un dataset mal preparado.

SemánticaLa etiqueta manda

Si una transformación cambia el significado, ya no estás ampliando datos: estás cambiando la tarea.

ContextoMás contexto, más cuidado

Los modelos contextuales pueden ayudar, pero también agregan latencia y costo de descarga en la primera ejecución.
