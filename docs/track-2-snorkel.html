Track 2 / Ruta Purista

Weak Supervision con Snorkel Modern Colab

Enseñar con reglas imperfectas: convertir criterio humano en señales de entrenamiento.

Abrir notebook en GitHub Abrir en Colab

Inspirado en Practical Natural Language Processing (O’Reilly), en practicalnlp.ai y en practical-nlp-code/Ch2. No existe afiliación oficial con esos recursos salvo que se indique explícitamente.

Qué vas a aprender

  • Cómo traducir criterio experto en labeling functions simples y revisables.
  • Cómo combinar reglas débiles para generar señales de entrenamiento útiles.
  • Cómo pensar weak supervision como apoyo al pipeline de datos, no como atajo mágico.
  • Cómo leer el costo técnico y el riesgo de sesgo de cada regla antes de usarla.

Por qué viene después de data prep y augmentación

Weak supervision funciona mejor cuando el dataset ya pasó por limpieza, tokenización y, si aplica, augmentación. Primero ordenás la materia prima; después agregás reglas que capturan criterio humano. Si saltás ese orden, terminás amplificando ruido, mezclando problemas de calidad y complicando la lectura de qué parte del pipeline realmente ayudó.

Cuándo ayuda y cuándo puede perjudicar

Ayuda cuando... Puede perjudicar cuando...
tenés pocas etiquetas humanas y necesitás arrancar con señales débiles; las reglas reflejan supuestos frágiles o demasiado específicos del equipo;
querés capturar conocimiento experto repetible sin etiquetar todo a mano; las labeling functions introducen ruido sistemático o sesgos difíciles de detectar;
querés acelerar una primera versión de dataset supervisado; la tarea exige alta precisión semántica y el error de regla rompe el objetivo.

Setup del entorno

  • La instalación en Colab puede ser lenta y sensible a versiones.
  • Si algo falla, reiniciá el runtime y volvé a correr la celda de setup desde cero.
  • Si aparece drift de dependencias, repetí la instalación limpia antes de seguir.
  • Usá las versiones sugeridas en el notebook si necesitás recuperar consistencia entre sesiones.

Antes de abrir el notebook

  • Leé la secuencia completa para entender dónde entra Snorkel en la ruta.
  • Recordá que Track 2 es más profundo y técnico, pero no “mejor” que Track 1: Módulo 1.
  • Usá Modern Colab como camino preferido; el notebook legacy queda solo como referencia histórica y comparación de fuente.
  • Revisá las reglas con ojo crítico: una labeling function también codifica una hipótesis.

Notas de clase

CriterioUna regla no es una verdad

Las reglas ayudan a empezar, pero siguen siendo aproximaciones humanas al problema.

SesgoLo que no mirás también enseña

Si una labeling function está mal diseñada, puede meter sesgo sistemático aunque parezca útil al principio.

SecuenciaPrimero datos, después reglas

Si la base está desordenada, weak supervision no corrige el problema: lo vuelve más difícil de diagnosticar.
