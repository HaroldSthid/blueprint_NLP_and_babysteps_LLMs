Web Scraping con BeautifulSoup

De una página web ruidosa a un dato útil.

Abrir notebook en GitHub Abrir en Colab

Inspirado en Practical Natural Language Processing (O’Reilly), en practicalnlp.ai y en practical-nlp-code/Ch2. No existe afiliación oficial con esos recursos salvo que se indique explícitamente.

Qué vas a aprender

  • Leer HTML con una mirada práctica.
  • Encontrar etiquetas, clases y nodos útiles para extraer texto.
  • Transformar una página desordenada en una tabla limpia para NLP.
  • Detectar cuándo el contenido necesita limpieza o validación extra.

Por qué importa para datasets de NLP

Antes de entrenar o evaluar modelos, alguien tiene que conseguir texto confiable. Web scraping ayuda a construir corpus, armar colecciones de referencia y detectar patrones de publicación que después afectan limpieza, deduplicación y calidad del dataset.

Antes de abrir el notebook

  • Leé primero la historia completa, no solo la primera celda.
  • Recordá que una web real cambia: lo que hoy funciona mañana puede romperse.
  • Si aparece HTML raro, no lo fuerces: inspeccioná estructura y contenido.
  • Verificá términos de uso, robots.txt y rate limits antes de recolectar datos.
  • Trabajá con paciencia: scraping es iteración, no magia.

Scraping responsable

  • Respetá los términos del sitio y su robots.txt.
  • Limitá la frecuencia de requests para no saturar el servidor.
  • Guardá solo lo que necesitás para el objetivo de aprendizaje o dataset.
  • Asumí que el HTML puede cambiar sin aviso y prepará el código para eso.

Notas de clase

HTMLLa estructura manda

BeautifulSoup no adivina la intención del sitio: trabaja sobre la estructura real. Si el selector falla, la página sigue ahí; cambió el mapa.

Calidad del datoExtraer no es limpiar

Obtener texto es solo el primer paso. Después hay que normalizar, filtrar ruido y decidir qué queda fuera del dataset.

Cambios del sitioUna página viva puede romper tu parser

Si el sitio cambia clases, encabezados o bloques repetidos, el scraper necesita revisión. El problema suele estar en el HTML, no en tu notebook.

ÉticaMás dato no siempre es mejor dato

Recolectar sin criterio puede empeorar el dataset. Elegí fuentes, límites y volumen con la misma disciplina que usás para modelar.
